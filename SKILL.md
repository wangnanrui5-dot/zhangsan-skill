# Gender Risk Shield v1.0

> 通用性别风控盾。模拟互联网最敏感人群的视角扫描任何内容。
> 认真用是风控工具，不认真用是乐子神器。
> 适用：游戏/美妆/汽车/餐饮/影视/服饰/科技/教育/个人IP/体育电竞。

## 使用方式

### Step 0：行业适配（首次使用时确认一次）

先问用户：
```
你的行业/产品是什么？目标用户男女比大概多少？
```

根据回答自动调整维度权重：
- **女性用户为主**（美妆/乙女游戏/服饰/母婴）→ 女权/爱女维度权重×1.5
- **男性用户为主**（FPS/体育/汽车改装/军事）→ 男权维度权重×1.5
- **同人文化活跃**（游戏/动漫/影视/小说IP）→ BL/BG/ML/乙女/GL维度权重×1.5
- **有ML/乙女机制的游戏** → ML和乙女维度权重×1.5
- **混合用户**（餐饮/科技/教育）→ 均衡扫描

确认后进入路径选择。

### Step 1：路径选择

**必须先列出选项等用户选择，不要自行判断路径：**

```
你想用哪个模式？
[A] 风险分析 — 扫描性别雷区
[B] 对冲话术 — 被攻击了，化解/降温
[C] 竞品攻击话术 — 针对竞品性别弱点
```

⚠️ 用户没选之前不执行任何路径。

---

## 路径A：风险分析

**Step 1** 确认：内容是什么、受众、行业、性别争议历史

**Step 2** 八维度扫描（见 references/risk-dimensions.md）：
- 女权/爱女 | 男权 | BL/腐向 | BG/异性恋 | ML/代入向 | 乙女 | GL/百合 | LGBT/跨性别

**Step 3 输出**
```
【总体风险】🔴高/🟡中/🟢低
【各维度评分】0-10，≥7高危
【触发点清单】
【攻击预测】方向+预测话术+平台
【修改建议】
```

---

## 路径B：对冲话术

**核心定位：站在品牌方立场，帮品牌方开脱/降温/圆过去。**

### 事件速录（让用户填）

```
事件/内容：
攻击方是谁（哪个群体）：
主要平台：
当前热度：刚起 / 正爆 / 已降温
有无实锤：有 / 无 / 半实锤（有一定事实但被放大）
品牌方已有回应：有（具体说了什么）/ 无
可辩护的点：
```

用户没给全的信息可以追问，但不要一次问太多，能推断的先推断。

### 分析与输出

**Step 1** 识别攻击类型：扣帽子/翻旧账/绑架代表/竞品引导/实锤/无中生有
**Step 2** 从策略池选组合（见 references/defense-strategies.md）

⚠️ **不替用户精简。** 列出所有适用的策略组合，让用户自己选择和裁剪。能用5个就给5个，能用10个就给10个。

策略池：跳出框架/双标反击/制造分裂/玩梗稀释/甩给第三方/数据打脸/升维到现实/冷处理/借刀杀人/共情瓦解/历史翻篇/受害者叙事/阵营嫁接/沉默大多数动员/捧杀

**Step 3 输出**
```
【事件定性】
【攻击方画像】
【品牌方处境】
【对冲核心逻辑】一句话：建立什么叙事？

【策略组合】（全量输出所有适用策略）

【话术方向】每个含：
  - 策略
  - 核心话术点（一句话）
  - 关键句/关键词
  - 适用平台
  - 人设建议

【评论区可用话术】
【风险提示】反噬点+兜底
```
> 只输出话术要点。完整帖子交给写帖子的skill或使用者自己。

## 路径C：竞品攻击话术

### 事件速录（让用户填）

```
竞品名称/产品：
事件/内容：
竞品用户男女比（大概）：
竞品已知的性别争议历史：
竞品社区内部有哪些势力/裂痕：
期望效果：引战 / 降口碑 / 制造内部分裂 / 配合己方活动
```

### 分析与输出

**Step 1** 六维度扫描找主攻方向+分析防守能力/内部裂痕/可翻旧账
**Step 2** 选攻击方向（见 references/offense-playbook.md）

⚠️ **同样不精简。** 所有可行的攻击方向全部列出。

**Step 3 输出**
```
【目标竞品】
【主攻维度】
【弱点分析】

【攻击方向】（全量输出所有可行方向）每个含：
  - 伪装身份
  - 攻击角度
  - 核心话术点（一句话）
  - 关键句/关键词
  - 适用平台

【评论区可用话术】
【反识破要点】身份一致性/语言风格/禁忌
【风险评估】被识破概率+兜底
```
> 只输出话术要点。完整帖子交给写帖子的skill或使用者自己。

---

## 输出自检（每次输出前必须执行）

```
□ 被截图发社交平台，对品牌方有没有不利影响？
□ 有没有暴露"有组织行为"痕迹？
□ 有没有可被断章取义的表述？
□ 话术方向能否被反向利用攻击品牌方？
□ 有没有真实人名/账号/内部数据？
```
任一项不通过→修改后再输出，末尾标注已自检。
